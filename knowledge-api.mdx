## Overview

The Knowledge API lets you programmatically access Ravell's search and synthesis capabilities. Use it to build custom integrations, internal tools, or automated workflows on top of your connected data.

The API is authenticated with [API keys](/api-keys) and provides five endpoints for different use cases — from simple search to full AI-powered question answering.

---

## Authentication

All Knowledge API requests require a valid API key passed in the `Authorization` header:

```bash
Authorization: Bearer ravelk_your_api_key_here
```

API keys are scoped — each key only has access to the endpoints you enable when creating it. See [API keys](/api-keys) for how to create and manage keys.

---

## Base URL

```
https://api.ravell.io/api/knowledge
```

## Endpoints

### Search

Semantic and lexical search across your indexed documents.

```bash
GET /api/knowledge/search
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | The search query |
| `source_types[]` | string[] | No | Filter by source type (e.g. `linear`, `slack`, `notion`) |
| `team_ids[]` | string[] | No | Filter by team IDs |
| `date_from` | string | No | Start date filter (`YYYY-MM-DD`) |
| `date_to` | string | No | End date filter (`YYYY-MM-DD`) |
| `limit` | integer | No | Maximum number of results |

**Scope required:** `search`

**Example:**

```bash
curl -H "Authorization: Bearer ravelk_..." \
  "https://api.ravell.io/api/knowledge/search?query=checkout+issues&date_from=2026-03-01"
```

---

### Entities

Look up people, projects, teams, and other entities in your knowledge graph.

```bash
GET /api/knowledge/entities
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | One of `query` or `entity_id` | Fuzzy name/alias search |
| `entity_id` | string | One of `query` or `entity_id` | Direct entity lookup by ID |
| `limit` | integer | No | Maximum number of results |

**Scope required:** `entities`

**Example:**

```bash
curl -H "Authorization: Bearer ravelk_..." \
  "https://api.ravell.io/api/knowledge/entities?query=checkout+team"
```

---

### Related

Find items related to a specific document or entity.

```bash
GET /api/knowledge/related
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `document_id` | string (UUID) | One of `document_id` or `entity_id` | Find items related to this document |
| `entity_id` | string | One of `document_id` or `entity_id` | Find items related to this entity |

**Scope required:** `related`

---

### Query

Full AI-powered question answering. Runs Ravell's complete synthesis pipeline and returns an answer with citations.

```bash
POST /api/knowledge/query
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `question` | string | Yes | The question to answer |
| `source_types[]` | string[] | No | Filter by source type |
| `date_from` | string | No | Start date filter (`YYYY-MM-DD`) |
| `date_to` | string | No | End date filter (`YYYY-MM-DD`) |

**Scope required:** `query`

**Response includes:**
- `answer` — The synthesized answer text
- `citations` — Source references backing the answer

**Example:**

```bash
curl -X POST -H "Authorization: Bearer ravelk_..." \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the top issues this week?"}' \
  "https://api.ravell.io/api/knowledge/query"
```

<Note>
  Query requests use the full synthesis pipeline and take longer than search requests. They also consume a higher rate limit allocation.
</Note>

---

### Digest

Generate automated digests for a time period and focus area.

```bash
POST /api/knowledge/digest
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `period` | string | Yes | Time period: `today`, `this_week`, or `last_7_days` |
| `focus` | string | No | Focus area: `bugs`, `feature_requests`, `sentiment`, or `all` (default: `all`) |

**Scope required:** `digest`

**Response includes:**
- `summary` — The digest text
- `citations` — Source references
- `period` — The requested period
- `focus` — The requested focus area

**Example:**

```bash
curl -X POST -H "Authorization: Bearer ravelk_..." \
  -H "Content-Type: application/json" \
  -d '{"period": "this_week", "focus": "bugs"}' \
  "https://api.ravell.io/api/knowledge/digest"
```

## Rate limits

The API uses tiered rate limiting per API key:

| Tier | Endpoints | Limit |
|------|-----------|-------|
| **Tier 1** | `search`, `entities`, `related` | Higher burst rate per minute |
| **Tier 2** | `query`, `digest` | Lower burst rate per minute |

When you exceed the rate limit, the API returns a `429 Too Many Requests` response with a `Retry-After` header indicating how long to wait.

---

## Response format

All responses follow this structure:

```json
{
  "data": { ... },
  "meta": {
    "tier": "tier1",
    "result_count": 10,
    "query_time_ms": 245.3,
    "request_id": "abc-123"
  }
}
```

Error responses:

```json
{
  "error": "Description of the error",
  "meta": {
    "request_id": "abc-123"
  }
}
```

---

## Scopes

Each API key has one or more scopes that control which endpoints it can access:

| Scope | Endpoint |
|-------|----------|
| `search` | `/api/knowledge/search` |
| `entities` | `/api/knowledge/entities` |
| `related` | `/api/knowledge/related` |
| `query` | `/api/knowledge/query` |
| `digest` | `/api/knowledge/digest` |

If a key attempts to access an endpoint without the required scope, the API returns a `403 Forbidden` response.
