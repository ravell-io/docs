## Terms

| Term | Definition |
|------|------------|
| **Ask** | A conversation thread where you ask questions and get evidence-backed answers with citations. Each Ask can have multiple follow-up messages. |
| **Agent** | A scheduled query that runs on a recurring basis (e.g. every Monday at 9am). Agents produce reports that appear in your Inbox. |
| **Citation** | A clickable reference in an answer that links to the original source data (Linear issue, Notion page, Intercom conversation, or Slack message). |
| **Context** | The central view combining your individual profile, workspace profile, and Memory. Used to personalize answers. |
| **Inbox** | Where proactive reports from your agents land. Each agent run produces a report you can read, follow up on, or push to Notion. |
| **Memory** | Facts Ravell learns about you over time from conversations and manual entries. Used to personalize answers across sessions. |
| **Profile (Individual)** | Your personal context: role, focus areas, current work, preferences. Auto-generated from your connected sources and editable manually. |
| **Profile (Workspace)** | Your team's context: company mission, goals, product context, ICP. Shared across all workspace members. |
| **Scoping** | Limiting a question to specific sources (e.g. "Linear only") or time ranges (e.g. "last 30 days") for faster, more focused answers. |
| **Slack Agent** | The interface for using Ravell in Slack via @Ravell mentions or the `/ravell` command. Supports questions, Linear actions, and agent management. |
| **Source** | A connected integration (Linear, Notion, Intercom, Slack, Attio) that Ravell syncs and indexes for search. |
| **Template** | A pre-built agent configuration with a question and schedule for common use cases like weekly roundups or standup prep. |
| **Visibility** | Whether an Ask or Agent is **private** (only you) or **shared** (visible to your workspace). |
| **API Key** | A token for programmatic access to Ravell via the API or MCP. Created and managed in Settings → API Keys. |
| **Entity** | A product concept (problem, feature request, or topic) automatically extracted from your sources and tracked in the product graph. |
| **MCP (Model Context Protocol)** | A protocol that connects Ravell to AI clients like Claude Desktop. Uses an API key and the `@ravell-io/mcp` package. |
| **Product Intelligence** | Ravell's automated problem-detection layer. Continuously analyzes connected sources to surface problems, feature requests, and trends. |
| **Push to Notion** | Exporting an answer or report from Ravell to a Notion database as a new page with formatted content and citations. |
| **Run** | A single execution of an Agent. Each run produces a report that appears in your Inbox. Runs have statuses: queued, running, success, error, or skipped. |
| **Weekly Brief** | An automated weekly summary of product intelligence (problems, emerging issues, trends) delivered to a Slack channel. |
| **Workspace** | Your team's shared space in Ravell. All sources, Asks, Agents, and reports live in a workspace. |
