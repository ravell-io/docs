## Overview

Queries are one-off or saved questions. The API supports creating queries, listing history, and managing favorites.

---

## Create query

`POST /api/v1/queries`

Creates and runs a query.

**Request body**:

```json
{
  "query": "What are the top open issues in our current cycle?",
  "source_ids": ["uuid1", "uuid2"]
}
```

- `query`: Required. The question text.
- `source_ids`: Optional. Limit search to specific integration IDs.

**Response**: Query object with `id`, `query`, `status`, `result`, etc.

---

## Get query

`GET /api/v1/queries/:id`

Returns a single query with its result and metadata.

---

## List queries

`GET /api/v1/queries`

Returns the user's queries. Supports pagination.

---

## Query history

`GET /api/v1/queries/history`

Returns query history for the current user.

---

## Favorites

`POST /api/v1/queries/:id/favorite`

Marks a query as favorite.

`DELETE /api/v1/queries/:id/unfavorite`

Removes favorite.

`GET /api/v1/queries/favorites`

Returns favorited queries.

---

## Cancel query

`POST /api/v1/queries/:id/cancel`

Cancels a running query.
