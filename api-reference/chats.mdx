## Overview

Chats are conversation threads for Asks. You create a chat, send messages (ask), and can follow up in the same thread. The API supports creating chats, asking questions, and retrieving history.

---

## Create chat

`POST /api/v1/chats`

Creates a new chat. Optionally include an initial message or title.

**Request body** (optional):

```json
{
  "title": "Optional initial title"
}
```

**Response**: Chat object with `id`, `title`, `slug`, `created_at`, etc.

---

## Ask (send message)

`POST /api/v1/chats/:id/ask`

Sends a message to the chat and streams or returns the answer.

**Request body**:

```json
{
  "message": "What are the top open issues in our current cycle?",
  "source_ids": ["uuid1", "uuid2"]
}
```

- `message`: Required. The question or follow-up.
- `source_ids`: Optional. Limit search to specific integration IDs.

**Response**: Answer with citations. May be streamed depending on client support.

---

## Get chat

`GET /api/v1/chats/:id`

Returns a single chat with its messages and metadata.

---

## List chats

`GET /api/v1/chats`

Returns the user's chats, optionally filtered. Supports pagination.

---

## Chat history

`GET /api/v1/chats/history`

Returns chat history for the current user, ordered by recent activity.

---

## Update chat

`PATCH /api/v1/chats/:id`

Updates chat metadata (e.g. title).

---

## Move chat

`PATCH /api/v1/chats/:id/move`

Moves a chat to a folder or changes its visibility.

**Request body**:

```json
{
  "visibility": "private",
  "query_folder_id": "uuid"
}
```

- `visibility`: Optional. `"private"` or `"shared"`.
- `query_folder_id`: Optional. UUID of the folder to move the chat into.

---

## Rewrite title

`POST /api/v1/chats/rewrite_title`

Uses AI to generate a title from query text. Useful for auto-titling new chats.

**Request body**:

```json
{
  "query_text": "What did customers say about checkout last month?"
}
```

**Response**: `{ "title": "...", "source": "llm" | "fallback" }`

---

## Get chat by slug

`GET /api/v1/chats/by_slug?slug=<slug>`

Returns a chat by its slug. Used for shared links (e.g. `https://app.ravell.io/app?chat=<slug>`). Requires workspace access.

---

## Delete chat

`DELETE /api/v1/chats/:id`

Deletes a chat and its messages.
