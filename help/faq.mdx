<AccordionGroup>
  <Accordion title="Why did Ravell give me an incomplete or inaccurate answer?">
    Ravell's answers depend on the data available in your connected sources. Common reasons for incomplete answers:

    - **Missing sources**: Connect more tools for broader coverage. The more sources, the richer the answers.
    - **Sync delay**: Data syncs periodically. Very recent changes may not be indexed yet. Trigger a manual sync from **Sources** if needed.
    - **Vague question**: Be specific with time ranges, teams, and topics. See [Better questions](/better-questions).
    - **Scoping too narrow**: If you scoped to one source, the answer might be in another. Try "all sources" first.

    Always check citations to verify — click any citation to see the original source data.
  </Accordion>

  <Accordion title="How fresh is the data?">
    Ravell syncs connected sources periodically. Most data is updated within minutes of changes in your tools. You can trigger a manual sync from **Sources** at any time.

    For time-sensitive questions, check the sync status in **Sources** to confirm data is up to date.
  </Accordion>

  <Accordion title="Can teammates see my Asks?">
    It depends on the visibility setting:

    - **Private**: Only you can see it.
    - **Shared**: Visible to all workspace members.

    You choose visibility when creating an Ask, and can change it later. See [Visibility and sharing](/visibility-sharing).
  </Accordion>

  <Accordion title="How do I get better answers?">
    - Be specific: include time ranges, teams, and topics
    - Use follow-ups instead of new Asks for related questions
    - Connect more sources for cross-tool context
    - Set up your profile for personalized results
    - Scope to specific sources when you know where the answer lives

    See the full guide: [Better questions](/better-questions).
  </Accordion>

  <Accordion title="Why is sync slow?">
    Initial sync can take several minutes for large workspaces with thousands of issues, pages, or conversations. Subsequent syncs are incremental and much faster.

    If sync seems stuck, check the sync status in **Sources**. Try disconnecting and reconnecting the integration if errors persist.
  </Accordion>

  <Accordion title="What happens if I disconnect a source?">
    Ravell stops syncing and removes that source's data from the index. Past Ask answers that cited that source still show the citations, but the underlying data is no longer available for new queries. You can reconnect at any time to re-sync.
  </Accordion>

  <Accordion title="How does Ravell use my data?">
    Your data is used only to answer your questions and generate reports. It is not used to train AI models. Data is stored encrypted, scoped to your workspace, and never shared across workspaces. See [Security & privacy](/security-privacy).
  </Accordion>

  <Accordion title="Can I use Ravell without connecting all sources?">
    Yes. Ravell works with even a single source. Connect the tool your team uses most and add others over time. The more sources you connect, the richer the cross-tool answers, but each source adds value on its own.
  </Accordion>

  <Accordion title="How do I use Ravell from Slack?">
    Mention **@Ravell** or use the `/ravell` slash command in any channel or DM. You can ask questions, create Linear issues, and manage agents — all in natural language. See [Slack Agent](/slack-agent).
  </Accordion>

  <Accordion title="Can I export answers?">
    Yes. You can push answers to Notion via the **Push to Notion** button, or use **Copy link** to share a direct URL with teammates. Agent reports can be delivered to Slack channels automatically.
  </Accordion>

  <Accordion title="How do I report a bug or request a feature?">
    Use the in-app feedback form:

    1. Click **Send feedback** in the sidebar.
    2. Select **Bug**, **Feature**, or **General**.
    3. Add a title and description, then click **Submit**.

    Your feedback automatically includes context like the current page URL and browser info to help us investigate faster. You can also email [support@ravell.io](mailto:support@ravell.io).
  </Accordion>
</AccordionGroup>
