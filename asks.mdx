## What are Web Asks?

**Web Asks** are the primary way to use Ravell in the app. Type a question in plain language, and Ravell searches your connected sources, retrieves relevant evidence, and synthesizes an answer with citations. Each Ask is a conversation thread: follow up to go deeper without repeating context.

Answers are grounded in your data — every claim ties back to a source. You always see where the information came from and can verify it yourself.

---

## What you can ask

| Category | Example questions |
|----------|-------------------|
| **Issue tracking** | "What are the top open issues in our current cycle?" · "Find ENG-142" · "Show me stale issues" |
| **Customer feedback** | "What did customers say about checkout last month?" · "Cluster feedback about pricing" |
| **Team activity** | "What did the backend team ship last week?" · "What's blocking the most issues?" |
| **Discussions** | "What did we discuss about the API last week?" · "Summarize #product this week" |
| **Documentation** | "Find the product roadmap in Notion" · "Summarize our onboarding docs" |
| **Cross-source** | "Connect customer feedback to our roadmap" · "What do we know about Acme Corp?" |
| **CRM** | "What meetings did we have with Acme?" · "Show recent notes on the Series B deal" |

For 50+ more examples, see the [Question library](/use-cases/question-library).

## Asking a question

1. Click **New Ask** in the sidebar, or use the main input when no chat is selected.
2. Type your question in plain language. Be as specific as you can — time ranges, topics, and source hints improve results.
3. Optionally set **visibility** (private or shared) and **scope** (all sources or specific ones).
4. Press Enter or click Send. The answer streams in with citations.

---

## Follow-ups

Reply in the same thread to ask follow-up questions. Ravell keeps the full conversation context, so you can say things like:

- "Which of those are high priority?"
- "Show me the Linear links for the top 3."
- "What did we decide about the checkout flow?"
- "Create a bug for that" — Ravell uses the most recent topic when you say "that" or "this".

You don't need to repeat the original question. Follow-ups can refine, narrow, or extend the answer.

---

## Citations

Every answer includes **citations** — clickable references to the source data. When you click a citation:

- You see the exact excerpt Ravell used.
- You can jump to the original item (Linear issue, Notion page, Intercom conversation, Slack message).

Citations let you verify answers and dig into the evidence yourself. If something seems off, check the source.

### Context used panel

The **Sources** rail (next to the answer) may show a "Context used" panel. This displays which profiles (individual and workspace) and Memory were used for the answer, their freshness, and any quality flags. Use **View context used** to jump to the Context view and see or edit your profiles.

---

## Scoping

By default, Ravell searches across all connected sources. You can narrow the search:

- **All sources** (default) — Linear, Notion, Intercom, Slack.
- **Specific source** — Use the source selector to limit (e.g. "Linear only" or "Notion only").

Scoping is useful when:
- You know the answer is in one tool.
- You want faster results (fewer sources = less to search).
- You're testing or comparing answers across tools.

---

## Visibility and sharing

When creating a new Ask, you choose **private** or **shared**:

- **Private** — Only you can see it. Good for exploratory questions or sensitive topics.
- **Shared** — Visible to your workspace. Teammates can open it from the sidebar. Good for collaboration and shared context.

You can change visibility later from the report menu (**Make private** / **Share with workspace**). Use **Copy link** to share a direct URL to an Ask.

## Actions from chat

You can take actions directly from a web Ask — not just get answers. When Linear or Notion is connected, Ravell can create and update records as part of the conversation.

### Linear actions

Ask Ravell to create or update Linear records as a follow-up:

| Action | Example prompt |
|--------|---------------|
| **Create issue** | "Create a bug for the login timeout we just discussed" |
| **Update issue** | "Mark ENG-123 as done" |
| **Create project** | "Create a project for the checkout redesign" |
| **Create initiative** | "Create an initiative for Q2 platform improvements" |
| **Create document** | "Create a Linear document summarizing these findings" |
| **Add comment** | "Add a comment on ENG-142 with this summary" |

Ravell uses conversation context to fill in details — you don't need to specify title, description, or team explicitly. It also checks for duplicates before creating issues.

### Push to Notion

When Notion is connected, you can **push** an answer to a Notion page:

1. Open an Ask with an answer.
2. Click **Push to Notion** on the answer card.
3. Choose the Notion page or database to add the report.
4. The answer (with citations) is added as content in Notion.

You can also ask Ravell directly: "Push this to Notion" or "Save these results to Notion" as a follow-up prompt. Ravell can create new Notion pages, append to existing pages, and manage databases through natural language.

<Tip>
All the same Linear and Notion actions available in the [Slack Agent](/slack-agent) also work from web Asks. Use follow-up prompts to act on your results without switching tools.
</Tip>

## Organizing Asks

### Folders

Create **folders** to group related Asks in the sidebar:

- Click the folder icon or use **New folder** in the sidebar.
- Give the folder a name and optional icon.
- Drag Asks into folders, or move them via the Ask menu.
- Folders can be **private** (only you) or **shared** (visible to the workspace). A chat's visibility must match its folder's visibility.
- Deleting a folder moves its Asks back to the top level — the Asks themselves are not deleted.

### Favorites

**Favorite** an Ask to bookmark it for quick access:

- Click the star icon on any Ask, or use the Ask menu.
- Add a custom name and optional notes.
- Access all favorites from the favorites section in the sidebar.
- Unfavoriting removes the bookmark but keeps the original Ask.

## How it works

<Accordion title="Behind the scenes">
When you submit a question, Ravell runs a multi-stage pipeline:

1. **Intent** — Classifies what you're asking (find issues, summarize feedback, compare trends) and resolves references like "Project X" or "ENG-142" to real items.
2. **Plan** — Breaks the query into sub-questions and selects which sources to search.
3. **Search** — Retrieves documents using semantic, lexical, structured, and graph-based search simultaneously.
4. **Cluster & rank** — Groups evidence by theme and re-ranks so the best evidence surfaces first.
5. **Synthesize** — Generates the answer, grounding each claim in the retrieved evidence.
6. **Verify** — Checks that claims match the evidence and marks confidence where needed.

You see this progress as **streaming thoughts** — short status updates that show what Ravell is doing. Simple questions get a quick pass; complex ones get deeper reasoning.
</Accordion>

## Tips for better answers

| Tip | Why it helps |
|-----|-------------|
| **Be specific** | "Checkout feedback in the last 30 days" beats "customer feedback" |
| **Use follow-ups** | Reply in the thread to narrow, expand, or act on results |
| **Scope when you know** | "From Linear only" is faster when you know the answer is there |
| **Connect more sources** | More data = richer, more accurate answers |
| **Include time ranges** | "Last week", "this cycle", "since January" improve relevance |

For a full prompt engineering guide, see [Better questions](/better-questions).

## Where Asks live

- **Sidebar → Asks** — List of your past and current conversations. Click one to reopen it.
- **Sidebar → New Ask** — Start a fresh conversation.
- **Sidebar → Folders** — Asks organized into folders you create.
- **Sidebar → Favorites** — Bookmarked Asks for quick access.
- **Cmd+K** — Open chat search to find an Ask by title.
- **Copy link** — Share a direct URL (e.g. `https://app.ravell.io/app?chat=<slug>`).
