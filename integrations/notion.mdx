## Overview

Connect **Notion** to Ravell to search across your Notion workspace. Ravell indexes pages and databases so you can ask questions like "What did we document about the checkout flow?" and get answers with citations. You can also **push reports to Notion** from Asks and Inbox.

---

## Connect Notion

1. Go to **Sources** or **Settings → Integrations**.
2. Click **Connect** next to Notion.
3. Sign in to Notion and authorize Ravell.
4. Select which pages or databases to sync (or sync the whole workspace).
5. Ravell starts syncing. Initial sync may take a few minutes for large workspaces.

---

## What you can ask

| Question | What Ravell does |
|----------|------------------|
| "What did we document about the checkout flow?" | Searches Notion pages for checkout-related content |
| "Find the product roadmap page" | Direct lookup of a specific document |
| "Summarize our onboarding documentation" | Synthesizes content from onboarding-related pages |
| "What specs were written last month?" | Finds recently created or updated spec documents |
| "What's in our competitive analysis database?" | Searches database entries for competitive content |
| "Find all pages mentioning [feature name]" | Full-text and semantic search across your Notion workspace |

---

## What gets synced

- **Pages**: Title, content, properties.
- **Databases**: Schema and entries.

Ravell uses this data to answer questions and provide citations back to Notion pages.

---

## Push to Notion

Push any Ask answer or Agent report to Notion as a new page:

<Steps>
  <Step title="Open the answer">
    Open an Ask or Inbox report with the content you want to push.
  </Step>
  <Step title="Click Push to Notion">
    Click **Push to Notion** on the answer card.
  </Step>
  <Step title="Choose destination">
    Select the parent page or database where the new page should go.
  </Step>
  <Step title="Confirm">
    Ravell creates the page with the report content and citations, formatted for Notion (headings, lists, links).
  </Step>
</Steps>

**Use cases for Push to Notion:**
- Share answers with your team as Notion docs
- Build a knowledge base of recurring agent reports
- Archive sprint reviews and weekly roundups
- Create meeting prep docs from Ravell answers

## Notion actions from Slack

When both Notion and Slack are connected, you can perform Notion actions directly from the Slack Agent using natural language. Ravell delegates these requests to a Notion sub-agent with full read/write capabilities.

### Create pages

- `@Ravell Create a Notion page for the checkout redesign spec`
- `@Ravell Add a page to [database] with title "Sprint 12 Review"`

### Update pages

- `@Ravell Update the onboarding doc with the new steps we discussed`
- `@Ravell Append a section to the API spec page`

### Create and update databases

- `@Ravell Create a Notion database for tracking feature requests`
- `@Ravell Add a "Priority" property to the feedback database`

### Comments

- `@Ravell Add a comment on the product roadmap page`

### Search

- `@Ravell Search Notion for competitive analysis`
- `@Ravell Find the onboarding documentation in Notion`

### Set default database

- `@Ravell Set [database] as the default for new reports`

Once set, future **Push to Notion** exports use this database automatically.

---

## Re-sync and reindex

Ravell syncs Notion data periodically. Trigger a manual **Reindex** or **Sync** from **Sources** to refresh data after adding new pages or updating content.
