## Overview

Connect **Notion** to Ravell to search across your Notion workspace. Ravell indexes pages and databases so you can ask questions like "What did we document about the checkout flow?" and get answers with citations. Beyond search, Ravell can **create and update** pages, databases, and comments — all from natural language.

---

## Connect Notion

1. Go to **Sources** or **Settings → Integrations**.
2. Click **Connect** next to Notion.
3. Sign in to Notion and authorize Ravell.
4. Select which pages or databases to sync (or sync the whole workspace).
5. Ravell starts syncing. Initial sync may take a few minutes for large workspaces.

---

## What you can ask

| Question | What Ravell does |
|----------|------------------|
| "What did we document about the checkout flow?" | Searches Notion pages for checkout-related content |
| "Find the product roadmap page" | Direct lookup of a specific document |
| "Summarize our onboarding documentation" | Synthesizes content from onboarding-related pages |
| "What specs were written last month?" | Finds recently created or updated spec documents |
| "What's in our competitive analysis database?" | Searches database entries for competitive content |
| "Find all pages mentioning [feature name]" | Full-text and semantic search across your Notion workspace |

## Actions you can take

Ravell supports bidirectional actions with Notion — you can create and update pages, databases, and comments directly from the Slack Agent or via follow-up prompts.

### Pages

| Action | Example |
|--------|---------|
| **Create page** | "Create a Notion page for the API spec" |
| **Create page in database** | "Create a page in the Product Specs database about the new checkout flow" |
| **Update page** | "Update the API spec page title" |
| **Archive page** | "Archive the old roadmap page" |
| **Append content** | "Add these meeting notes to the standup page" |

### Databases

| Action | Example |
|--------|---------|
| **Create database** | "Create a Notion database for tracking feature requests" |
| **Update database** | "Rename the Tasks database to Sprint Tasks" |
| **Set default database** | "Set my default Notion database to Product Specs" |

### Comments

| Action | Example |
|--------|---------|
| **Add comment** | "Add a comment on the roadmap page about the deadline change" |

### Push to Notion

| Action | Example |
|--------|---------|
| **Push report** | "Push this to Notion" or "Save that answer to Notion" |
| **Export to database** | "Export this report to my Product Specs database" |

<Tip>
  When you set a default database, future exports automatically go there unless you specify another destination.
</Tip>

## From Slack

All Notion actions work from the Slack Agent. Mention **@Ravell** with your request:

- `@Ravell Create a Notion page for the API spec`
- `@Ravell Push that to Notion`
- `@Ravell Add a comment on the roadmap page`
- `@Ravell Set my default Notion database to Product Specs`
- `@Ravell Create a database for tracking bugs`

Ravell uses thread context, so you can discuss something in Slack and then say `@Ravell Push this to Notion`. See [Slack Agent](/slack-agent) for full details.

## What gets synced

- **Pages**: Title, content, properties.
- **Databases**: Schema and entries.

Ravell uses this data to answer questions and provide citations back to Notion pages.

## Push to Notion (from the app)

Push any Ask answer or Agent report to Notion as a new page:

<Steps>
  <Step title="Open the answer">
    Open an Ask or Inbox report with the content you want to push.
  </Step>
  <Step title="Click Push to Notion">
    Click **Push to Notion** on the answer card.
  </Step>
  <Step title="Choose destination">
    Select the parent page or database where the new page should go.
  </Step>
  <Step title="Confirm">
    Ravell creates the page with the report content and citations, formatted for Notion (headings, lists, links).
  </Step>
</Steps>

**Use cases for Push to Notion:**
- Share answers with your team as Notion docs
- Build a knowledge base of recurring agent reports
- Archive sprint reviews and weekly roundups
- Create meeting prep docs from Ravell answers

## Re-sync and reindex

Ravell syncs Notion data periodically. Trigger a manual **Reindex** or **Sync** from **Sources** to refresh data after adding new pages or updating content.
