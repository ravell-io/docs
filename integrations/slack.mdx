## Overview

Slack serves two roles in Ravell:

1. **Slack as a source**: Ravell indexes selected channels so you can search Slack content in Asks (e.g. "What did we discuss about the API last week?").
2. **Slack Agent**: Use Ravell directly in Slack via @Ravell or `/ravell` — ask questions, create Linear issues, and manage agents without leaving Slack.

---

## Connect Slack

1. Go to **Sources** or **Settings → Integrations**.
2. Click **Connect** next to Slack.
3. Sign in to Slack and authorize Ravell.
4. Select which **channels** to index. Only messages from selected channels are searchable.
5. Ravell starts syncing. Initial sync may take a few minutes.

---

## Slack as a source (indexing)

### What gets synced

- **Messages**: From the channels you selected.
- **Threads**: Thread replies are included for context.

Ravell uses this data to answer questions about discussions, decisions, and context shared in Slack.

### Which channels should I index?

Index channels where important discussions happen:

- **Product channels**: #product, #roadmap, #feature-requests
- **Engineering channels**: #engineering, #backend, #frontend
- **Customer-facing channels**: #customer-feedback, #support
- **Decision-making channels**: #leadership, #strategy

You don't need to index social or high-noise channels (#random, #watercooler). Start with 3-5 key channels and expand as needed.

---

## Slack Agent (interface)

The Slack Agent is a separate capability from indexing. It lets you interact with Ravell directly in Slack:

- **Ask questions**: `@Ravell What are the top open issues?` — searches all connected sources, not just Slack.
- **Create Linear records**: `@Ravell Create a bug about login timeouts` — natural language actions.
- **Manage agents**: `@Ravell Create an agent that runs every Monday summarizing top issues` — create, pause, resume, delete.

The Slack Agent is available in any channel or DM where the Ravell app is installed. See [Slack Agent](/slack-agent) for full details and examples.

---

## Agent delivery

You can deliver Agent reports to Slack channels automatically:

1. Create or edit an agent.
2. Set visibility to **Shared with workspace**.
3. Enable **Deliver to Slack** and select the channel.
4. When the agent runs, the report is posted to the selected channel with citations.

See [Agents](/agents) for details.
