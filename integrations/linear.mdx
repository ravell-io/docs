## Overview

Connect **Linear** to Ravell to search across your issues, projects, cycles, initiatives, and documents. Ravell doesn't just read Linear — it can also **create and update** issues, projects, initiatives, documents, customer needs, and more.

---

## Connect Linear

1. Go to **Sources** or **Settings → Integrations**.
2. Click **Connect** next to Linear.
3. Sign in to Linear and authorize Ravell.
4. Select which **teams** to sync. You can sync all teams or choose specific ones.
5. Ravell starts syncing. Initial sync may take a few minutes.

---

## What you can ask

| Question | What Ravell does |
|----------|------------------|
| "What are the top open issues in our current cycle?" | Searches issues in the active cycle, ranked by priority |
| "What did the Payments team ship last week?" | Lists completed issues for that team |
| "Find ENG-142" | Direct lookup of a specific issue |
| "What bugs were filed vs resolved this week?" | Compares created vs completed bug issues |
| "What's the status of the checkout redesign project?" | Finds and summarizes the project |
| "Which issues are assigned to me?" | Uses your profile to filter by assignee |
| "What initiatives are in progress?" | Lists active initiatives |
| "Show me stale issues with no activity for 2 weeks" | Finds issues with no recent updates |
| "Compare velocity across teams this cycle" | Cross-team completion analysis |
| "What customer needs are linked to this project?" | Surfaces customer context from Linear |

## Actions you can take

Ravell supports bidirectional actions with Linear — you can create and update records directly from the Slack Agent or via follow-up prompts.

### Issues

| Action | Example |
|--------|---------|
| **Create issue** | "Create a bug about login timeouts in the Backend team" |
| **Update issue** | "Update ENG-123 to done" or "Change the priority of ENG-456 to urgent" |

### Projects

| Action | Example |
|--------|---------|
| **Create project** | "Create a project for the checkout redesign" |
| **Update project** | "Update the checkout project status to in progress" |

### Initiatives

| Action | Example |
|--------|---------|
| **Create initiative** | "Create an initiative for Q2 platform improvements" |
| **Update initiative** | "Update the Platform initiative status" |

### Documents

| Action | Example |
|--------|---------|
| **Create document** | "Create a Linear document for the API spec" |
| **Update document** | "Update the API spec document with the latest decisions" |

### Customer data

| Action | Example |
|--------|---------|
| **Create customer** | "Create a customer record for Acme Corp" |
| **Create customer need** | "Log this as a customer need on ENG-123" |
| **Add comment** | "Add a comment on ENG-142 summarizing the discussion" |
| **Add attachment** | "Add this link as an attachment to ENG-142" |

<Tip>
  Ravell checks for duplicates before creating issues and uses conversation context when you refer to "this" or "that".
</Tip>

## From Slack

All Linear actions work from the Slack Agent. Mention **@Ravell** with your request:

- `@Ravell Create a bug about login timeouts`
- `@Ravell Update ENG-123 to done`
- `@Ravell Create a project for the checkout redesign`
- `@Ravell Log this as a customer request on ENG-123`

Ravell uses thread context, so you can discuss an issue in Slack and then say `@Ravell Create a ticket for that`. See [Slack Agent](/slack-agent) for full details.

## What gets synced

| Data type | What's included |
|-----------|----------------|
| **Issues** | Title, description, status, assignee, labels, project, cycle, initiative, comments |
| **Projects** | Name, description, status, milestones |
| **Cycles** | Name, dates, status, associated issues |
| **Initiatives** | Name, description, status |
| **Documents** | Title, content |
| **Customer needs** | Linked feedback and feature requests |

---

## Team selection

When connecting, you choose which Linear teams to sync. Only data from selected teams is indexed. Update team selection anytime in **Settings → Integrations → Linear**.

### Team categories

After selecting teams, you can categorize each team as either:

- **Internal work** — Teams focused on internal engineering, infrastructure, or operations.
- **Customer needs** — Teams that handle customer-facing work, feedback, or support requests.

These categories affect how Ravell classifies and retrieves documents. For example, issues from customer-needs teams are weighted differently when answering questions about customer feedback versus internal engineering work.

---

## Re-sync and reindex

Ravell syncs Linear data periodically. You can trigger a manual **Reindex** or **Sync** from **Sources** to refresh data immediately after changes.
