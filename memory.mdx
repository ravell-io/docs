## What is Memory?

**Memory** is Ravell's store of what it learns about you over time. When you ask questions, have conversations, or share context, Ravell can remember key facts (e.g. "You work on the checkout team" or "You prefer summaries in bullet points"). This helps personalize future answers across sessions.

**Why it matters:** Facts you share are extracted and stored. Next time, they're already in context. For each question, Ravell pulls the most relevant memories; your role and identity are always included. The more you use Ravell, the better it understands you and the less you need to repeat yourself.

---

## What Memory stores

Ravell organizes memories into categories so they can be applied in the right context:

| Category | What it captures | Example |
|----------|-----------------|---------|
| **Role & profile** | Your role, team, seniority | "You are a Senior PM on the Checkout team" |
| **Projects** | Current and past projects you work on | "You own the checkout redesign project" |
| **Preferences** | How you like answers formatted, level of detail | "You prefer bullet-point summaries" |
| **Product** | Product facts, features, technical details | "The mobile app uses React Native" |
| **Internal workflow** | Team processes, tools, rituals | "Your team uses 2-week sprints starting Mondays" |
| **Company & business** | Company context, competitors, market | "Main competitor is Acme Corp" |
| **Roadmap** | Upcoming plans, priorities, milestones | "v2 launch is planned for Q3" |

Each memory also has a **confidence score** (how certain Ravell is about the fact) and an **importance score** (how relevant it is to your work). Higher-importance memories are more likely to be included in context when you ask a question.

---

## View and edit Memory

1. Go to **Settings → Memory**.
2. You'll see what Ravell has stored about you.
3. You can **edit** or **delete** individual items.
4. Use **Add** to explicitly tell Ravell something to remember.

---

## Clear Memory

If you want to start fresh:

1. Go to **Settings → Memory**.
2. Click **Clear memory**.
3. Confirm. Ravell will stop using stored facts for personalization until you provide new context.

## Suggested memories to add

Manually adding a few key facts makes a big difference. Here are some ideas:

| Memory | Why it helps |
|--------|-------------|
| "I work on the checkout team" | Scopes answers to your team's work |
| "I prefer bullet-point summaries" | Adjusts output format |
| "Our main competitor is [name]" | Improves competitive analysis |
| "I'm responsible for the mobile app" | Prioritizes mobile-related issues |
| "We use 2-week sprints starting on Mondays" | Aligns time-based questions with your cadence |

### Teaching Ravell your vocabulary

If your team uses specific terms, acronyms, or project names, add them as memories:

- "When I say 'Phoenix', I mean the checkout redesign project"
- "PDP means Product Detail Page"
- "CSM stands for Customer Success Manager"

This helps Ravell resolve references in your questions more accurately.

## How memories are created

Memories come from two sources:

- **You add them manually** — Go to **Settings → Memory** and add a fact explicitly. These are marked as **user-created** and given the highest confidence.
- **Ravell extracts them automatically** — After a conversation, Ravell analyzes what you said and extracts key facts (up to 5 per conversation). These are marked as **auto-extracted** and start with a lower confidence score that increases as they are corroborated.

### Verification

Memories have a verification status that evolves over time:

| Status | Meaning |
|--------|---------|
| **Unverified** | Newly extracted, not yet confirmed |
| **User-asserted** | You created or edited this memory manually |
| **Corroborated** | Multiple conversations or sources support this fact |
| **Contradicted** | New information conflicts with this memory — it is excluded from future answers |

When Ravell detects a contradiction (e.g. you said "I work on Checkout" but later say "I moved to the Platform team"), it marks the old memory as contradicted and creates a new one.

### Web enrichment

When you mention companies, products, or other entities Ravell doesn't know about, it may search the web to fill in context. For example, if you mention a competitor by name, Ravell can look up what they do and store that as a memory for future questions. Web-enriched memories are marked with a moderate confidence score.

## Enable or disable Memory

You can turn Memory on or off per workspace:

1. Go to **Settings → Memory**.
2. Toggle **Memory enabled** on or off.

When disabled, Ravell stops extracting new memories and does not use existing ones for personalization. Your stored memories are preserved — re-enable Memory any time to resume.

---

## Safety and privacy

Ravell applies safety filters before storing any memory:

- **No secrets** — API keys, tokens, passwords, and credentials are blocked automatically.
- **No PII** — Social security numbers, credit card numbers, and phone numbers are filtered out.
- **No prompt injection** — Attempts to inject instructions into memory are detected and rejected.

Memory is scoped to your account. Workspace members cannot see your personal memories. You can delete any memory at any time from **Settings → Memory**.
